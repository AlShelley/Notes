oh-my-zsh配置

下载安装

sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

修改主题

nano ~/.zshrc 或者 nano /home/lonelyor/.zshrc ZSH_THEME="agnoster"

查看可使用的主题列表 ls ~/.oh-my-zsh/themes

更换默认shell

chsh -s /usr/bin/zsh 重启即可看到效果

安装插件

增加插件直接写名字就行，用空格隔开。以下是一个示例。 plugins=(git autojump)

安装文件夹跳转工具 autojump， sudo pacman -S autojump

把以下代码添加到~/.zshrc才能使 autojump 生效 [[ -s ~/.autojump/etc/profile.d/autojump.sh ]] && . ~/.autojump/etc/profile.d/autojump.sh

安装zsh-syntax-highlighting命令高亮插件 git clone https://github.com/zsh-users/zsh-syntax-highlighting.git echo "source ${(q-)PWD}/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh" >> ${ZDOTDIR:-$HOME}/.zshrc

安装命令自动补全zsh-autosuggestions插件 git clone git://github.com/zsh-users/zsh-autosuggestions $ZSH_CUSTOM/plugins/zsh-autosuggestions

一键解压，自带的 extract

配置~/.zshrc 文件，添加内容： plugins=(git autojump zsh-autosuggestions extract zsh-syntax-highlighting)

请务必保证插件顺序，zsh-syntax-highlighting必须在最后一个。 然后在文件的最后一行添加：source ~/.oh-my-zsh/custom/plugins/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh

最后使用 source ~/.zshrc 命令让配置生效。

安装 powerline字体 pacman -S powerline-fonts

部署和启动 mysql in docker

1、首先安装 docker，这里略过

2、安装 mysql 的 docker 镜像docker pull mysql:5.7.13

3、启动 mysql，这里的 name 可以自行指定。

docker run -d -p 127.0.0.1:6606:3306 \
--name open_source_mysqldb \
-e MYSQL_ROOT_PASSWORD=root \
mysql:5.7.13

4、查看已经启动的 docker 镜像，这里注意 name 后面要用到。docker ps -a

5、进入容器bashdocker exec -it open_source_mysqldb bash

6、进入 mysql 并创建数据库和账号权限配置 mysql -h localhost -P 6606 -u root -p 

mysql> CREATE DATABASE IF NOT EXISTS vuldb DEFAULT CHARSET utf8 COLLATE utf8_general_ci;
mysql> grant all on vuldb.* to vuluser@'%' identified by 'vulpassword';
mysql> flush privileges;
mysql> quit

7、退出容器 bashexit

Mac 上的 svnGUI 工具：smartsvn 安装、激活和使用

smartsvn 是 mac 上的 svn 客户端，svn 是类似 git 的版本控制工具。

1、下载 smartsvn srmatsvn 官网下载地址 2、在任意目录下新建文件 smartsvn.license，然后写入以下内容：

Name=csdn  
Address=1337 iNViSiBLE Str.  
Email=admin@csdn.net  
FreeUpdatesUntil=2099-09-26  
LicenseCount=1337  
Addon-xMerge=true  
Addon-API=true  
Enterprise=true  
Key=4kl-<Zqcm-iUF7I-IVmYG-XAyvv-KYRoC-xlgsv-sSBds-VAnP6

3、在安装软件需要导入 license 时选择上述文件或者安装软件后在菜单栏的 help-register 中导入密钥文件即可激活软件。

4、点击菜单栏的project-checkout，添加远程目录，然后输入认证信息，输入本地目录信息，即可挂载远程目录到本地。

5、若认证出错，则点击菜单栏的 SmartSVN-preferences，然后选择 Authentication，使用 delete 删除之前存储的认证信息，然后重新 checkout 即可。

Mac 终端下 svn 常用命令

Mac 可以不用安装 svnGUI 而直接使用命令进行 svn 操作，常见svn 命令如下： svn help 查看命令帮助 svn checkout xxx 拉取远程目录到本地 svn add file 往版本库中添加新文件

更多常用命令可参考：Mac终端下svn常用命令

通过命令行操作进行认证后，其认证信息存储在 ~/.subversion/auth/svn.simple目录下，若认证失败，需要在此目录下情况认证信息后重新进行认证。

怎么解决"该虚拟机似乎正在使用中"问题

在用VMware虚拟机的时候，有时会发现打开虚拟机时提示“该虚拟机似乎正在使用中。如果该虚拟机未在使用，请按“获取所有权(T)”按钮获取它的所有权。否则，请按“取消(C)”按钮以防损坏。配置文件: D:\win10x64\Windows 10 x64.vmx。”这是由于虚拟机未正常关闭引起的，下面看看解决办法。

1.首先点击“获取所有权(T)”按钮，会发现弹出一个窗口，显示“无法打开虚拟机: D:\win10x64\Windows 10 x64.vmx获取该虚拟机的所有权失败。主机上的某个应用程序正在使用该虚拟机。配置文件: D:\win10x64\Windows 10 x64.vmx。”等信息。这说明获取所有权失败了。

2.当点击“取消(C)”按钮是直接退出窗口，什么也不弹出。这时发现不管点那个按钮都不能解决问题。

3.点击“开始”菜单，选择“文件资源管理器”。

4.打开“文件资源管理器”后，进入到存放此台Vmware虚拟机虚拟磁盘文件及配置文件存放的位置（也就是弹出提示窗口上的路径），在这里是D:\win10x64这个目录下，找到后缀为.lck文件夹。

5.然后，将后缀为.lck的文件夹删除，或者或避免删错可以移动到备份文件夹中，也可以直接重命名此文件夹，在这里将文件夹Windows 10 x64.vmx.lck改为Windows 10 x64.vmx.lck.backup。

6.然后重新打开虚拟机，发现此虚拟机可以正常打开了，这样就成功的解决了“该虚拟机似乎正在使用中”问题。

以后记得正常关机。
